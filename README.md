# NBA 2K Rating Predictor

Проект для анализа статистики игроков NBA и предсказания их рейтинга в NBA 2K.

## Что делает проект

Проект:

- загружает датасет с игроками NBA;
- удаляет лишние колонки;
- строит несколько графиков для анализа данных;
- обучает модели линейной регрессии;
- сравнивает качество моделей по RMSE;
- показывает, какие признаки сильнее всего связаны с рейтингом.

## Данные

Источник данных:

```text
https://www.kaggle.com/datasets/willyiamyu/nba-2k-ratings-with-real-nba-stats
```

Датасет лежит здесь:

```text
data/nba_rankings_2014-2020.csv
```

Целевая колонка (target):

```text
rankings
```

То есть модель пытается предсказать рейтинг игрока в NBA 2K.

## Основные признаки

В датасете есть игровые статистики игроков NBA:

- `AGE` — возраст игрока;
- `GP` — количество сыгранных игр;
- `W` — победы;
- `L` — поражения;
- `MIN` — минуты за игру;
- `PTS` — очки за игру;
- `FGM` — забитые броски с игры;
- `FGA` — попытки бросков с игры;
- `FG%` — процент попаданий с игры;
- `3PM` — забитые трехочковые;
- `3PA` — попытки трехочковых;
- `3P%` — процент трехочковых;
- `FTM` — забитые штрафные;
- `FTA` — попытки штрафных;
- `FT%` — процент штрафных;
- `OREB` — подборы в нападении;
- `DREB` — подборы в защите;
- `REB` — подборы за игру;
- `AST` — передачи;
- `TOV` — потери;
- `STL` — перехваты;
- `BLK` — блок-шоты;
- `PF` — фолы;
- `DD2` — дабл-даблы;
- `TD3` — трипл-даблы;
- `+/-` — показатель плюс-минус.

## Что удаляется из признаков

Из модели убраны:

- `Unnamed: 0` — технический индекс;
- `PLAYER` — имя игрока;
- `rankings` — target;
- `SEASON` и `TEAM` — категориальные текстовые признаки;
- `FP` — fantasy points, потому что это само по себе является некоторой моделью.

## Качество данных

После загрузки данных проверяются пропуски и дубликаты.

В датасете:

- пропущенных значений нет;
- полных дубликатов строк нет;
- рейтинг находится в диапазоне от 62 до 98.

Также отдельно проверяются большие ошибки модели. После удаления самых больших ошибок качество модели почти не меняется, поэтому это больше используется как проверка выбросов.

## Графики

В проекте строятся:

- распределение рейтингов;
- корреляционная матрица;
- график `PTS vs rankings`;
- boxplot рейтингов;
- важность признаков Ridge;
- распределение ошибок модели;
- сравнение моделей по RMSE.

## Модели

Используются:

- baseline-модель, которая предсказывает средний рейтинг;
- обычная линейная регрессия;
- линейная регрессия после масштабирования признаков;
- линейная регрессия с подобранным `alpha`;
- Ridge после удаления самых больших ошибок на обучении.

## Как запустить

Создать виртуальное окружение на Windows:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

Создать виртуальное окружение на Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить проект:

```bash
python main.py
```

## Основная идея

Рейтинг NBA 2K сильнее всего связан с базовой статистикой игрока:

- очки;
- попадания с игры;
- подборы;
- передачи;
- минуты;
- победы;
- блок-шоты;
- перехваты.
